HTML

A linguagem de construção web

Criada pelo britânico Tim Berners-Lee, o acrônimo HTML significa HiperText Markup Language, traduzindo ao português: Linguagem de Marcação de Hipertexto. HTML é o bloco de construção mais básico da web, esse compõe a maior parte das páginas da internet e dos aplicativos online. Sua função é o significado e a estrutura do conteúdo da web. Por conta disso, o HTML não é considerado uma linguagem de programação, já que ele não pode criar funcionalidades dinâmicas. Ao invés disso, com o HTML, os usuários podem criar e estruturar seções, parágrafos e links usando elementos, tags e atributos.

O "Hipertexto" presente no significado do seu nome, refere-se aos links que conectam páginas da Web entre si, seja dentro de um único site ou entre sites. Links são um aspecto fundamental da web. Ao carregar conteúdo na Internet e vinculá-lo a páginas criadas por outras pessoas, você se torna um participante ativo na World Wide Web (www).

Também vale notar que o HTML agora é considerado um padrão oficial da internet. O World Wide Web Consortium (W3C) mantêm e desenvolve especificações do HTML, além de providenciar atualizações regulares.

Apesar do HTML estar muito presente no desenvolvimento web, só fazendo o uso dele não será possível estruturar um site com todos os requisitos para que ele seja funcional e estético. Por conta disso, existem diversas linguagens de desenvolvimento web, como o CSS, responsável pela formatação dos itens e Java Script, responsável pelo controle de dados.




Para o que serve o HTML

  • Desenvolvimento web. Os desenvolvedores usam códigos HTML para projetar como um navegador vai exibir os elementos das páginas, como textos, hiperlinks e arquivos de mídia.

  • Navegação na internet. Os usuários podem navegar facilmente e inserir links entre páginas e sites relacionados, já que o HTML é amplamente usado para incorporar hiperlinks.

  • Documentação. O HTML torna possível a organização e a formatação de documentos, de maneira similar ao Microsoft Word.

  • Estruturação de Conteúdo: O HTML é usado para estruturar o conteúdo de uma página da web em elementos como cabeçalhos, parágrafos, listas, links, imagens e muito mais. Ele define a hierarquia e a organização do conteúdo, tornando-o legível e acessível.

  • Hipertexto: O HTML permite a criação de links (âncoras) que conectam diferentes páginas da web e recursos, permitindo a navegação entre elas. Isso é fundamental para a experiência de navegação na web

  • Incorporação de Mídia: HTML permite a incorporação de vários tipos de mídia, como imagens, áudio e vídeo, em uma página da web, tornando-a mais interativa e rica em conteúdo.

  • Formulários: HTML oferece elementos para criar formulários interativos, permitindo que os usuários enviem informações para o servidor web. Isso é essencial para recursos como caixas de pesquisa, formulários de contato e processamento de dados de entrada do usuário. Como o presente nesse repositório do GitHub.

  • Semântica: HTML fornece elementos semânticos que descrevem o significado do conteúdo, tornando-o mais acessível para pessoas com deficiência e melhorando a indexação por mecanismos de busca.

  • Compatibilidade com Navegadores: HTML é suportado por todos os principais navegadores da web, tornando-o uma escolha confiável para criar conteúdo acessível em diferentes plataformas e dispositivos.

  • Integração com outras tecnologias: HTML pode ser combinado com CSS para controlar a aparência visual das páginas e com JavaScript para adicionar interatividade e funcionalidade avançada às páginas da web.

  • Acessibilidade: HTML oferece recursos e práticas recomendadas para tornar o conteúdo web acessível a pessoas com deficiência, garantindo que todos possam usar e interagir com as páginas da web.

Como funciona o HTML
Um site médio inclui diversas páginas HTML diferentes. Por exemplo, ele pode trazer uma página inicial, uma página “sobre” e uma página de contato. Cada uma delas possui um arquivo HTML separado.

Documentos HTML são arquivos que terminam com uma extensão .html ou .htm. Um navegador lê o arquivo HTML e renderiza o seu conteúdo para que os usuários da internet possam vê-lo.

Todas as páginas HTML possuem uma série de elementos, que consistem num conjunto de tags e atributos. Os elementos HTML são os tijolos de construção de uma página da internet. Uma tag diz para o navegador onde um elemento começa e termina, enquanto um atributo descreve as características de um elemento.

As três principais partes de um elemento são:
  • Tag de abertura – usada para dizer onde um elemento começa a ter efeito. A tag é cercada de colchetes angulares para abertura e fechamento <>. Por exemplo, use a tag de abertura 'p' para criar um parágrafo.

  • Conteúdo – essa é a parte que os usuários verão.

  • Tag de fechamento – igual à tag de abertura, mas com uma barra antes do nome do elemento. Por exemplo, '/p' para encerrar um parágrafo.


Outra parte crucial de um elemento HTML é o seu atributo, que possui duas seções — um nome e um valor de atributo. O nome identifica a informação adicional que um usuário deseja acrescentar, enquanto o valor de atributo fornece mais especificações. HTML e suas tags
  • html: Define o início e o fim do documento HTML e contém todo o conteúdo da página.

  • head: Contém informações sobre o documento, como o título da página, links para folhas de estilo e metadados.

  • title: Define o título da página que aparece na guia do navegador ou na barra de título.

  • meta: Fornece informações sobre o documento, como o conjunto de caracteres usado e outras metainformações.

  • link: Usado para vincular a folhas de estilo externas (CSS) à página.

  • style: Define estilos CSS diretamente no documento HTML.

  • script: Permite a inclusão de código JavaScript na página.

  • body: Contém todo o conteúdo visível da página, como texto, imagens, links e outros elementos.

  • h1, h2, h3, h4, h5, h6: São cabeçalhos que indicam a hierarquia de títulos na página, do h1 (o mais importante) ao h6 (o menos importante).

  • p: Define um parágrafo de texto.

  • a: Cria links para outras páginas ou recursos da web.

  • img: Exibe imagens na página.

  • ul: Define uma lista não ordenada.

  • ol: Define uma lista ordenada.

  • li: Define um item de lista em ul ou ol.

  • div: É uma divisão genérica usada para agrupar elementos e aplicar estilos.

  • span: É uma marcação genérica usada para aplicar estilos a partes específicas do texto.

  • table: Cria uma tabela para exibir dados tabulares.

  • tr: Define uma linha em uma tabela.

  • td: Define uma célula de dados em uma tabela.

  • form: Cria um formulário para coletar dados do usuário.

  • input: Define campos de entrada em um formulário, como caixas de texto, botões de rádio e caixas de seleção.

  • button: Cria um botão que pode ser usado em formulários.

  • textarea: Define uma área de texto de várias linhas em um formulário.

  • label: Rótulo associado a um elemento de formulário para melhorar a acessibilidade.

  • iframe: Permite a incorporação de conteúdo de outras páginas ou recursos da web.

  • audio e video: Permitem a incorporação de áudio e vídeo na página.

  • canvas: Usado para desenhar gráficos e animações na página usando JavaScript.

  • header, nav, main, section, article, aside, footer: São elementos de estruturação semântica usados para criar uma estrutura mais significativa na página, facilitando a compreensão do conteúdo

  • hr: Cria uma linha horizontal para separar seções de conteúdo.

Prós do HTML
  • Uma linguagem amplamente utilizada com diversos recursos e uma comunidade gigante;

  • Roda em todos os navegadores;

  • Fácil aprendizado;

  • Totalmente grátis e de código aberto;

  • Marcações limpas e consistentes;

  • Os padrões oficiais da internet são mantidos pelo Consórcio World Wide Web (W3C);

  • Integração fácil com linguagens de back-end como PHP e Node.js.

Contras do HTML
  • Geralmente utilizado para páginas estáticas. Para funcionalidade dinâmicas você provavelmente vai precisar usar JavaScript ou outra linguagem de back-end como PHP;

  • Não permite a implementação de lógica. Por isso todas as páginas precisam ser criadas separadamente, mesmo se utilizarem os mesmos elementos, como cabeçalhos e rodapés;

  • Alguns navegadores demoram para adotar novos recursos;

  • O comportamento do navegador é algumas vezes difícil de prever (ex.: navegadores antigos muitas vezes não renderizam novas tags).
